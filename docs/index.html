Features

从写入恢复到读路径优化

WAL 预写日志

写入请求先追加到 `current.wal`，数据库重启时重放未刷盘记录，提供基础恢复能力。

MemTable 与 SSTable

内存中使用有序表组织数据，达到阈值后刷盘为不可变 SSTable，并维护索引与 key 范围。

Bloom Filter

SSTable 打开时加载过滤器，查询不存在的 key 时减少无效磁盘访问。

LRU Cache

缓存热点 key 的 value，降低重复读取 SSTable 的成本。

后台 Compaction

合并多个 SSTable，只保留最新版本并清理删除标记，降低读放大和磁盘占用。

工程化工具

提供交互式 CLI、本地 benchmark、Makefile、CMake 和核心流程测试。

Quick Start

本地构建与运行

构建与测试

git clone git@github.com:CozyOct1/MiniKV.git
cd MiniKV
make -j
make test

启动 CLI

./build/minikv_cli ./data
put name minikv
get name
flush
compact
stats
